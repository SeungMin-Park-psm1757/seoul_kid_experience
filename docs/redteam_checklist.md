# 레드팀 체크리스트

## 질문 1. 이거 그냥 예약 목록 예쁘게 보여주는 것 아닌가요?

### 방어 논리
아닙니다. 본 작품은 개별 프로그램 추천이 아니라, 자치구·연령대·주차별 선택가능도와 플랜B 격차를 시각화합니다. 핵심은 “목록”이 아니라 “선택권의 공간·시간 격차”입니다.

## 질문 2. 실시간 예약 가능 여부를 정확히 반영하나요?

### 방어 논리
공개 데이터 범위 안에서 서비스상태, 운영기간, 접수기간, 이용대상 등을 기준으로 탐색적 지표를 구성했습니다. 실시간 잔여석 분석은 본 작품의 범위가 아니며, 향후 예약시스템 연계 시 고도화 가능합니다.

## 질문 3. 노쇼 데이터를 분석했나요?

### 방어 논리
노쇼 발생률 분석은 하지 않습니다. 공개 데이터로 방어가 어렵기 때문입니다. 대신 취소·노쇼·신청 제한 규칙의 가시성과 복잡성을 이용장벽 요소로 다룹니다.

## 질문 4. 지표 가중치는 자의적인 것 아닌가요?

### 방어 논리
초기 탐색적 지표로서 부모의 실제 선택조건인 연령, 주말, 비용, 접근성, 상태, 다양성을 반영했습니다. 제출본에는 가중치 민감도 또는 한계도 명시합니다.

## 질문 5. 정책 기여도는 무엇인가요?

### 방어 논리
서울시는 이 시각화를 통해 빈 주말이 반복되는 자치구, 플랜B가 부족한 지역, 특정 연령대가 배제되는 블라인드 스팟을 확인할 수 있습니다. 이는 주말 공공체험 공급, 예약 안내 개선, 인접구 연계 추천 정책의 근거가 됩니다.

## 질문 6. 왜 웹 대시보드인가요?

### 방어 논리
공고상 시각화 결과물은 대시보드/인포그래픽 등을 포함합니다. 본 결과물은 예약·회원관리 서비스가 아니라, 필터와 상호작용을 통해 공공데이터를 탐색하게 하는 인터랙티브 시각화입니다.
