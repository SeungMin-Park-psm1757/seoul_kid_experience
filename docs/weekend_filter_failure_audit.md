# 주말 필터 실패 진단

작성 시각: 2026-05-13

## 1. 현재 추천 로직의 날짜 처리 방식

기존 추천 로직은 `date_start`와 `date_end`를 중심으로 선택 날짜 또는 같은 주말 포함 여부를 판단했다. 이 방식은 하루짜리 프로그램에는 비교적 안전하지만, 실제 API의 장기 운영 프로그램에는 위험하다.

공공서비스예약 데이터에는 `04.22~11.26`처럼 전체 운영 기간이 들어가고, 실제 회차 요일은 프로그램명이나 시간 문구에 `수, 목 오후 13:00~15:00`처럼 별도로 적히는 사례가 있다. 기존 로직은 이 운영 요일을 최우선으로 보지 않아 주말 선택 시 평일 전용 프로그램이 추천 후보에 섞였다.

## 2. `date_start/date_end`만으로 발생하는 오류

`date_start~date_end`는 “예약 또는 운영 기간”일 수 있으며, “선택한 날짜에 실제 운영한다”는 뜻이 아니다.

예를 들어 `2026-04-22~2026-11-26 / 수, 목 오후 13:00~15:00` 프로그램은 2026년 5월 16일 토요일이 기간 안에 포함되더라도 실제 회차는 수요일과 목요일이다. 따라서 토요일 추천, 추천 후보 5곳, 대체 후보, 지도 기본 결과에 노출되면 안 된다.

## 3. 요일 파싱이 필요한 이유

실제 운영 요일은 다음 필드에 흩어져 있을 수 있다.

- `title`: 프로그램명 안의 `(수, 목 오후 13:00~15:00)`, `(금 오후 13:00~14:30)`, `(평일주간)`
- `time_text`: 문화행사 시간 또는 회차 시간
- `target_text`: 대상 또는 참여 안내 안의 요일 단서
- `rule_summary` 또는 `description`: 운영 규칙, 회차 안내
- `raw_*`: 원본 API 필드의 `SVCNM`, `PRO_TIME`, `DATE`, `ETC_DESC`

따라서 추천 가능 여부는 “기간 포함 여부”가 아니라 “선택 날짜 요일과 명시 운영 요일의 일치 여부”를 먼저 판단해야 한다.

## 5. 문제 집계 방식

수정 전 기준으로 2026-05-16 토요일, 7세, 노원구 출발 조건에서 기본 추천 후보가 112개까지 생성될 수 있었다. 이 중 프로그램명 또는 시간 문구에 `수, 목`, `금 오후`, `평일`, `평일주간`, `평일야간`, `주중` 등 명확한 평일 신호가 있는 항목이 Top 5와 대체 후보에 들어갈 수 있었다.

최종 집계는 새 일정 파서를 적용한 뒤 `docs/weekend_filter_validation_report.md`에 기록한다.

## 7. 테스트 계획

- 수·목 운영 프로그램은 토요일 추천에서 제외한다.
- 금요일 오후 운영 프로그램은 토요일 추천에서 제외한다.
- 평일주간/평일야간 문구가 있는 프로그램은 주말 추천에서 제외한다.
- 토요일/주말/상설/화~일 운영 프로그램은 조건이 맞으면 유지한다.
- 오늘의 추천, 추천 후보 5곳, 대체 후보, 지도 기본 결과가 모두 동일한 일정 하드필터를 사용한다.
- 제외된 평일 항목은 “지금은 추천하지 않은 이유”에서만 제한적으로 설명한다.
