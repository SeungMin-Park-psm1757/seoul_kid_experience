# 사용자 의사결정 흐름

## 1. 첫 화면에서 답을 얻는 흐름

부모 사용자는 첫 화면에서 아이 나이, 출발 자치구, 날짜, 이동시간, 비용, 실내/실외, 관심분야를 조정한다. 앱은 바로 상단에 “이번 주말 바로 확인할 수 있는 후보 N개”를 문장으로 보여준다.

핵심 판단 순서:

- 오늘의 결론에서 지금 신청 가능, 이번 주말 가능, 무료/저비용, 마감/확인 필요 수를 확인한다.
- “지금 신청 가능한 곳” 카드 3~5개를 먼저 본다.
- 각 카드의 “왜 갈 만한가?”와 “주의” 문구를 확인한다.
- 공식 예약/출처 링크에서 실제 신청 가능 여부를 최종 확인한다.

## 2. 마감 후보를 보고 플랜B를 찾는 흐름

마감, 접수종료, 확인 필요 후보는 추천 카드와 분리되어 “예약이 어렵거나 확인이 필요한 곳”에 표시된다. 부모는 인기 있어 보이는 후보가 왜 기본 추천에서 빠졌는지 확인하고, 동일 화면에서 플랜B를 본다.

플랜B 기준:

- 원 후보와 동일 프로그램 제외
- 같은 날짜 또는 ±7일 우선
- 같은/유사 연령대
- 같은/유사 카테고리
- 같은 자치구 또는 인접 자치구 우선
- 예약 가능/상시/마감임박 후보 우선
- 무료/저비용 및 사용자의 실내/실외 조건 선호

## 3. 조건 완화로 후보 부족 이유를 이해하는 흐름

후보가 부족할 때 사용자는 “조건을 조금 풀면 후보가 얼마나 늘어날까?” 퍼널을 본다. 날짜 ±7일, 인접 자치구, 유료 포함, 실외 포함, 관심분야 전체 허용 순으로 후보 수 증가를 확인한다.

## 4. 정책 인사이트로 넘어가는 흐름

부모용 판단이 끝난 뒤 정책 인사이트 탭에서 자치구별 선택가능도, 플랜B 부족, 연령별 블라인드 스팟, 이용장벽을 본다. 정책 탭은 첫 화면이 아니라 후반부에 배치해 부모의 30초 의사결정을 방해하지 않는다.

## 5. API 모드에서 확인할 것

상단 배지가 `실제 API 데이터`이면 서울시 OpenAPI 수집 결과를 기반으로 한다. `샘플 데이터`이면 제출 전 API 연결과 수집 검증이 필요하다. `데이터 모드 오류`가 보이면 샘플과 API 데이터가 섞인 상태이므로 추천을 신뢰하지 말고 캐시를 비운 뒤 다시 수집해야 한다.

본 결과물은 서울시 OpenAPI와 공개 출처 링크를 기반으로 후보를 탐색하는 시각화 도구입니다. 실제 회차별 잔여석과 최종 신청 가능 여부는 공식 예약/출처 페이지에서 확인해야 합니다.
