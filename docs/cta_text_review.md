# CTA 문구 검토

작성일: 2026-05-13

## 좋은 CTA 기준

- 행동이 명확하다.
- 클릭 후 기대 결과가 분명하다.
- 짧다.
- 모바일에서 읽기 쉽다.
- 예약을 대행하는 것처럼 오해시키지 않는다.

## CTA 점검표

| 현재 문구 | 문제 | 추천 문구 | 클릭 후 동작 |
|---|---|---|---|
| 추천 결과 보기 | 현재 가장 명확함 | 유지 | 현재 필터 기준으로 추천 결과를 다시 계산하고 부모용 홈 결과를 보여줌 |
| 조건 초기화 | 명확함 | 유지 | 필터와 이미 가본 곳 제외 목록을 기본값으로 복원 |
| 이미 가본 곳 제외 설정 | 약간 길지만 의미 명확 | 유지 또는 `가본 곳 제외하기` | 페이지 맨 아래 설정 섹션을 펼침 |
| 공식 링크에서 확인 | 예약 대행처럼 보이지 않아 적절 | 유지 | 공식 예약/출처 페이지를 새 탭으로 열기 |
| 이 후보 상세 보기 | 명확함 | 유지 | 상세 + 대체 후보 탭에서 볼 후보로 선택 |
| 상세 보기 | 짧고 명확함 | 유지 | 카드 후보를 상세 후보로 선택 |
| 대체 후보 보기 | 명확함 | 유지 | 선택 후보 기준 대체 후보 확인 |
| 대체 후보 | 버튼으로는 약간 짧음 | `대체 후보 보기` | compact 카드에서 대체 후보 확인 |
| 이미 가본 곳으로 표시 | 길지만 오해가 적음 | 유지 | 현재 세션 제외 목록에 프로그램 추가 |
| 제외하기 | 무엇에서 제외되는지 모호함 | `가본 곳으로 표시` | 현재 세션 제외 목록에 프로그램 추가 |
| 왜 제외됐는지 보기 | 신청 어려운 후보용으로 적절 | 유지 | 제외 이유가 있는 상세 후보로 선택 |
| 이 설정으로 다시 추천 보기 | 명확함 | 유지 | 이미 가본 곳 제외 조건을 반영해 추천 재계산 |
| 전체 지도 | 명사형 버튼 | `전체 지도 보기` | 추천 후보 지도 표시 |
| 추천하지 않은 이유 | 명사형 버튼 | `추천하지 않은 이유 보기` | 제외/마감/불일치 후보 이유 표시 |
| 조건 넓히기 | 짧고 행동적 | 유지 | 조건 변경 시 후보 수 변화 표시 |
| API/원천 데이터 다시 수집 | 부모용에는 기술적 | 심사용이면 유지 | API 또는 fallback 데이터를 다시 빌드 |
| 샘플 데이터로 재생성 | 심사용/개발자용 | 유지 | 샘플 데이터 빌드 |
| 데이터 다시 빌드 | 오류 상황에서 적절 | 유지 | 처리 데이터 재생성 |

## 우선 수정 후보

1. `제외하기` → `가본 곳으로 표시`
2. `전체 지도` → `전체 지도 보기`
3. `추천하지 않은 이유` → `추천하지 않은 이유 보기`
4. `대체 후보` → `대체 후보 보기`

위 문구는 앱 기능 확정 후 일괄 반영하는 것이 좋다.
