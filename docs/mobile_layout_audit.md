# 모바일 레이아웃 감사

작성일: 2026-05-13

## 1. 현재 상단 영역 구성

현재 상단은 Streamlit 기본 앱 컨테이너 안에 메인 질문형 제목, 설명 문구, 데이터 상태 박스, 데이터 확인 expander, 핵심 필터 패널 순서로 구성되어 있다. 이전 화면에는 영문 장식 헤더와 큐레이션 배지가 있었으나, 제목 잘림을 막기 위해 사용자 화면에서 제거하는 방향으로 정리한다.

## 2. 제목이 잘리는 원인 후보

- Streamlit 기본 header가 실제 콘텐츠 영역 위를 덮거나, 브라우저 캡처 영역에서 콘텐츠의 맨 위가 잘려 보일 수 있다.
- `.block-container`의 상단 padding을 계속 늘리는 방식은 화면별로 다르게 보이며 근본 해결이 아니다.
- 제목 위 영문 장식 헤더가 남아 있으면 좁은 화면에서 제목 위치를 밀거나 잘려 보이게 한다.
- line-height가 충분하지 않거나 제목 컨테이너에 overflow 관련 속성이 있으면 글자가 잘릴 수 있다.
- 모바일 safe-area를 고려하지 않으면 앱 내 브라우저 또는 모바일 브라우저 상단 UI와 겹칠 수 있다.

## 3. 관심분야 필터가 잘리는 원인

- 관심분야 멀티선택을 4열 grid 안에 넣으면 선택 칩이 많을 때 높이가 부족해진다.
- 일부 BaseWeb select 내부 컨테이너가 고정 높이처럼 보이며 칩이 여러 줄로 늘어날 때 잘릴 수 있다.
- 데스크톱 중심 column 배치가 모바일 360~430px 폭에서 자연스럽게 쌓이지 않으면 가로/세로 잘림이 생긴다.

## 4. 사용자가 클릭할 수 있는 요소 목록

- 핵심 필터 입력: 아이 나이, 학년/단계, 출발 자치구, 날짜, 이동시간, 비용, 실내/실외, 관심분야
- 필터 하단 CTA: 추천 결과 보기, 조건 초기화, 이미 가본 곳 제외 설정
- 추천 결과 빠른 선택: 오늘의 추천 보기, 추천 후보 5곳, 대체 후보, 추천하지 않은 이유, 전체 지도, 전체 보기
- 후보 카드 CTA: 공식 링크에서 확인, 상세 보기, 대체 후보 보기, 이미 가본 곳으로 표시
- 데이터 확인 expander: API/원천 데이터 다시 수집, 샘플 데이터로 재생성

## 5. 클릭 후 이동해야 할 목적지

- 추천 결과 보기: 부모용 홈의 추천 결과 영역을 갱신하고 오늘의 추천을 먼저 보여준다.
- 조건 초기화: 필터 기본값을 복원하고 추천 결과를 다시 계산한다.
- 이미 가본 곳 제외 설정: 하단 설정 섹션을 펼치고 안내 메시지를 보여준다.
- 상세 보기: 상세 + 대체 후보 탭에서 볼 후보를 선택 상태로 저장한다.
- 대체 후보 보기: 선택 후보를 기준으로 상세 탭의 대체 후보를 확인하도록 선택 상태를 저장한다.
- 이미 가본 곳으로 표시: 현재 세션의 제외 목록에 프로그램을 추가하고 추천 결과를 다시 계산한다.

## 6. 현재 빠져 있는 CTA 목록

- 필터 변경 후 명확한 기본 CTA
- 이미 가본 곳 제외 설정을 실제로 여는 버튼
- 설정 완료 후 추천 재계산 버튼
- 카드별 “이미 가본 곳으로 표시” 버튼
- 빠른 이동 패널의 실제 클릭 가능한 선택 UI

## 7. 수정 계획

1. 영문 장식 헤더, 큐레이션 배지, 불필요한 상단 장식을 사용자 화면에서 제거한다.
2. Streamlit header와 겹치지 않도록 기본 header를 투명/비표시 처리하고 safe-area padding을 적용한다.
3. 필터는 모바일에서 1열, 태블릿에서 2열, 데스크톱에서 최대 4열로 자연스럽게 쌓이게 한다.
4. 관심분야는 항상 전체 폭 영역으로 둔다.
5. 필터 하단에 추천 결과 보기, 조건 초기화, 이미 가본 곳 제외 설정 버튼을 배치한다.
6. 이미 가본 곳 제외는 페이지 맨 아래 섹션에 두고, 버튼으로 펼침 상태를 제어한다.
7. 추천 결과 상단에는 빠른 선택 버튼/라디오를 두어 다음 행동을 명확히 한다.
8. 모든 카드 하단에 공식 링크, 상세, 대체 후보, 제외 CTA를 배치한다.
