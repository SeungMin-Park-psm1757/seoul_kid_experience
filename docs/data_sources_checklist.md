# 데이터 소스 체크리스트

실제 데이터 수집 시 아래 순서로 확보한다. 모든 데이터는 가능하면 서울 데이터 허브 또는 서울 열린데이터광장 출처를 명시한다.

## 최소 실행 조합

### 1. 서울시 공공서비스예약 정보
- 용도: 실제 예약형 체험 후보 추출
- 기대 필드: 서비스명, 자치구/지역, 장소, 이용대상, 접수기간, 이용기간, 서비스상태, 요금, URL, 좌표
- 주의: 실시간 잔여석이 없을 수 있음. 실시간성 주장 금지.

### 2. 서울시 문화행사 정보
- 용도: 비예약형 또는 행사형 대체안 확보
- 기대 필드: 행사명, 장소, 자치구, 날짜, 이용대상, 이용요금, 행사시간, 좌표, URL

### 3. 서울형 키즈카페 시설현황정보
- 용도: 가족 친화적 실내 대체 시설 레이어
- 기대 필드: 시설명, 자치구, 주소, 이용정원, 운영일, 좌표, 연락처/URL

### 4. 서울시 문화공간/박물관·미술관 정보
- 용도: 예약 실패 시 fallback 가능한 상시 체험 공간
- 기대 필드: 시설명, 분류, 자치구, 주소, 관람료, 운영시간, 좌표, 홈페이지

### 5. 자치구별 아동 인구 또는 연령별 인구 통계
- 용도: 아동 1천 명당 선택가능도 등 정책 인사이트 분모
- 기대 필드: 자치구, 연령대 또는 나이, 인구 수

## 선택 보강 데이터

### 6. 자치구 경계 GeoJSON/SHP
- 용도: 실제 choropleth 지도 제작
- 없으면 패키지의 tilemap으로 대체 가능

### 7. 대중교통/도보 접근성 데이터
- 용도: 공간 접근성 고도화
- 1차 MVP에서는 자치구/인접구 기준으로 대체 가능

## 데이터 수집 후 작성할 문서

`docs/data_audit.md`에 다음 내용을 반드시 작성한다.

| 항목 | 내용 |
|---|---|
| 데이터명 | 공식 데이터셋 이름 |
| 출처 | URL 또는 포털명 |
| 갱신일 | 가능하면 최신 갱신일 |
| 주요 필드 | 분석에 사용한 컬럼 |
| 결측/제약 | 누락값, 좌표 없음, 실시간성 한계 등 |
| 사용 목적 | 지표/시각화와 연결 |
