# 데이터 한계와 정확성 원칙

## 핵심 원칙

이 앱은 예약 서비스가 아니라 공개 데이터 기반 선택지원 시각화다. 사용자는 후보를 비교한 뒤 공식 예약/출처 링크에서 실제 신청 가능 여부와 상세 규정을 최종 확인해야 한다.

## 공식 API 상태와 실제 예약 가능 여부의 차이

공공 API의 `접수중`, `예약가능`, `상시이용` 같은 상태는 탐색 우선순위를 정하는 데 사용한다. 다만 회차별 정원, 잔여석, 기관별 운영 변경, 현장 사정은 API 갱신 주기와 다를 수 있다. 따라서 화면에서는 “지금 신청 가능”이라고 그룹화하더라도 “공식 예약 페이지에서 최종 확인” 문구를 함께 표시한다.

## 부모용 추천에서 제외하는 데이터

부모용 추천은 단순 점수 정렬이 아니라 하드필터를 먼저 적용한다. 청년, 성인, 시니어, 반려동물, 조직 모집, 자원봉사, 강사·활동가 모집, 평일 전용 프로그램은 기본 추천과 대체 후보에서 제외한다. `제한없음` 또는 0~99세로 파싱된 항목도 자동 적합으로 보지 않고, 제목·대상·분류에서 아동·가족·초등 맥락이 확인될 때만 추천 후보로 사용한다.

## 잔여석을 확정하지 않는 이유

회차별 잔여석이 공개 API에서 일관된 필드로 제공되지 않을 수 있다. 공개 상세 페이지에 정원이나 잔여석 문구가 보이더라도 파싱 결과는 참고 정보일 뿐, 실시간 확정값으로 보장할 수 없다.

## 상세 페이지 파싱의 한계

상세 페이지 파싱은 공개 HTML 문구를 대상으로 하는 best-effort 보조 기능이다. 페이지 구조 변경, 네트워크 오류, 접근 제한, 문구 누락이 있으면 실패할 수 있다. 실패 시 앱은 후보를 삭제하지 않고 “확인 필요”로 표시한다.

## 이동시간 한계

현재 이동시간은 대중교통 API 기반 경로 시간이 아니다. 출발 자치구와 목적 자치구 사이의 상대거리 기반 추정 범위이며, 화면에는 `추정 40~50분`, `이동 난이도`처럼 표시한다. 실제 이동시간은 지도/교통 서비스에서 확인해야 한다.

## 예약 난이도 신호의 한계

예약 난이도는 공개 데이터의 현재 상태, 접수 문구, 날짜 여유, 상시·현장 여부를 기반으로 만든 참고 신호다. 반복 스냅샷이 충분히 쌓이기 전에는 과거 마감 패턴으로 표현하지 않는다.

## 샘플 데이터 사용 시 주의

`SEOUL_OPENAPI_KEY`가 없거나 API 호출이 실패하면 앱은 샘플 데이터만 사용한다. 샘플 데이터와 실제 API 데이터는 섞지 않으며, 데이터 모드는 `sample` 또는 `api`로 명확히 표시한다.

## API 수집본 사용 시 주의

2026-05-13 기준 live pipeline은 성공한 서비스명만 사용한다. 공공서비스예약 일부 후보 서비스명은 서버 오류 또는 필수 인자 누락으로 실패할 수 있으므로, 성공/실패 내역은 `docs/api_connection_report.md`에서 확인한다. 실패한 서비스명은 성공한 것처럼 처리하지 않는다.

본 결과물은 서울시 OpenAPI와 공개 출처 링크를 기반으로 후보를 탐색하는 시각화 도구입니다. 실제 회차별 잔여석과 최종 신청 가능 여부는 공식 예약/출처 페이지에서 확인해야 합니다.
