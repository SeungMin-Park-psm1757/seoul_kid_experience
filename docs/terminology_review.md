# 용어 검토 사전

작성일: 2026-05-13

## 목적

앱 내부 구현·정책 분석에는 유용하지만 부모용 화면에서는 딱딱하거나 기술적으로 보일 수 있는 용어를 분리한다. 최종 문구 반영 전, 외부 AI와 사용자가 아래 표를 기준으로 표현을 검토한다.

## 부모용 화면에서 피할 용어

- 플랜B
- 조건 완화 퍼널
- 데이터 품질 패널
- 선택가능도
- 하드필터
- 저마찰
- 상시/현장형
- API 확인
- 링크 확인 필요
- 후보 329개처럼 큰 후보 수를 첫 화면에서 강조하는 표현
- 정책 인사이트
- 우선 점검 매트릭스
- 제외 후보
- 예약 난이도 신호
- 대체안 sufficient / gap 등 영문 잔재
- Top 5, Top 1 등 영어식 순위 표현

## 용어 변환 표

| 내부 용어 | 부모용 표현 | 정책/심사용 표현 | 비고 |
|---|---|---|---|
| 플랜B | 막혔을 때 갈 만한 곳 / 대체 후보 | 대체 후보 충분도 | 부모 화면에서는 “대체 후보” 선호 |
| 조건 완화 퍼널 | 조건을 넓히면 늘어나는 후보 | 조건별 후보 손실 분석 | 차트명은 쉬운 말 우선 |
| 데이터 품질 패널 | 데이터 확인 | 데이터 품질 지표 | 부모용에는 “품질”보다 “확인” |
| 선택가능도 | 갈 수 있는 후보 수 | 선택가능도 | 정책 탭에서는 유지 가능 |
| 하드필터 | 조건에 맞지 않아 제외 / 조건에 맞는 후보 | 하드필터 | 부모 화면 노출 금지 |
| 저마찰 | 신청 부담이 낮은 후보 / 확인할 일이 적은 후보 | 이용장벽 낮음 | “저마찰”은 UX 내부어 |
| 상시/현장형 | 신청 부담이 낮은 후보 / 바로 확인하기 쉬운 후보 | 상시·현장 가능 후보 | 원천 데이터 해석에는 유지 가능 |
| API 확인 | 공공데이터 기준 | API 확인 | 부모 화면은 API보다 공공데이터 |
| 링크 확인 필요 | 공식 링크에서 확인 | 원문 확인 필요 | 불안감을 줄이는 표현 |
| 제외 후보 | 지금은 추천하지 않은 이유 | 제외 사유 분석 | “쓰레기통”처럼 보이지 않게 |
| 예약 난이도 신호 | 신청 부담 / 확인할 점 | 예약 난이도 신호 | 부모 화면에는 신호라는 말 최소화 |
| 우선 점검 매트릭스 | 사용하지 않음 | 우선 점검 매트릭스 | 정책 탭 전용 |
| 정책 인사이트 | 서울시 관점 / 지역별 기회 보기 | 정책 인사이트 | 부모 첫 화면에는 미노출 |
| Top 1 | 오늘의 추천 1곳 | 대표 추천 | 영어 줄이기 |
| Top 5 | 추천 후보 5곳 | 상위 후보 | 영어 줄이기 |
| 후보 329개 | 조건에 맞는 후보 더보기 | 전체 후보 수 | 첫 화면에서 큰 숫자 강조 금지 |
| gap | 부족 / 차이 | 격차 | 영문 잔재 제거 |
| sufficient | 충분 | 충분도 | 영문 잔재 제거 |

## 현재 화면에서 이미 순화한 표현

- `플랜B` → `대체 후보`, `첫 후보가 어려울 때 볼 곳`
- `하드필터 통과 후보` → `조건에 맞는 후보`
- `저마찰` → `신청 부담 낮음`
- `조건 완화 퍼널` → `조건을 넓히면 늘어나는 후보`
- `데이터 품질 패널` → `데이터 확인`
- `Top 5` → `추천 후보 5곳`

## 추가 검토가 필요한 표현

- `신청 부담 낮은 대체 후보`: 의미는 맞지만 다소 딱딱할 수 있다.
- `바로 확인할 수 있는 곳`: 예약 가능처럼 오해될 가능성은 낮지만, “확인”의 의미가 애매할 수 있다.
- `제외된 부적합 후보`: 부정적이므로 “조건과 맞지 않는 후보”가 더 안전하다.
- `노쇼 규칙`: 부모용 상세 화면에서는 “불참/미참여 규칙” 검토.
- `정책 지표 연령대`: 정책 탭 전용이지만 심사용 표현으로 다소 딱딱하다.
