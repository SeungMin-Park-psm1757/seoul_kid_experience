# 부모용 MVP 고도화 수정 계획

## 현재 구현 상태

- 화면 구조는 `부모용 홈`, `추천 탐색`, `후보 비교`, `상세 + 플랜B`, `정책 인사이트` 5탭으로 구성되어 있다.
- 부모용 홈에는 아이 나이, 학년, 출발 자치구, 날짜, 이동시간, 비용, 실내/실외, 관심분야 필터가 있다.
- 추천 결과는 프로그램 카드로 노출되며, 지도 탐색, 후보 비교, 상세/플랜B, 정책 차트가 이어진다.
- 데이터는 주로 `data/processed/unified_programs.csv`, `district_age_metrics.csv`, `weekly_choice_metrics.csv`를 사용한다.
- 현재 샘플 데이터는 더미명과 먼 미래 날짜가 섞여 있어 "이번 주말" 의사결정 도구로는 설득력이 약하다.

## 수정할 문제 목록

- 숫자가 붙은 더미 프로그램명을 실제 서울 공공데이터 스타일 명칭으로 교체한다.
- 날짜 매칭을 `exact_weekend`, `near_weekend`, `month_extension`, `future_reference`로 구분한다.
- 기본 추천에는 30일 초과 미래 후보를 포함하지 않는다.
- 접수 예정 후보가 예약 가능/상시 운영 후보보다 먼저 나오지 않도록 정렬한다.
- 이동시간을 정확한 교통시간처럼 보이지 않게 `추정 40~50분`, `자치구 간 상대거리 기반`으로 표현한다.
- 추천 이유와 주의 문구를 카드, 비교표, 상세 화면에 표시한다.
- 조건 완화 퍼널을 추가하고 PPT용 PNG를 저장한다.
- 정책 인사이트에 자동 요약 문장과 차트별 해석 문장을 추가한다.
- 데이터 품질 패널을 추가해 샘플/실제 데이터 여부와 필드 품질을 설명한다.
- 실제 서울 데이터 CSV/JSON 교체를 위한 normalize/import 문서를 보강한다.

## 수정 대상 파일

- `app.py`: 부모용 화면 정보 구조, 날짜/상태 정렬, 카드, 지도, 비교, 상세, 정책 인사이트, 품질 패널 개선
- `src/sample_data.py`: 샘플 프로그램명, 장소, 상태, 날짜 분포, 신규 필드 생성 개선
- `src/normalize.py`: 신규 필수 컬럼 보장, 실제 데이터 매핑 보강
- `src/config.py`: 통합 컬럼 정의 확장
- `src/build_all.py`: 샘플 재생성 및 문서/그림 저장 흐름 유지
- `docs/data_import_guide.md`: 실제 데이터 교체 가이드 추가
- `docs/ppt_capture_guide.md`: PPT 이미지/캡처 가이드 추가
- `docs/limitations_and_redteam.md`: 한계와 정확성 리스크 문서화
- `docs/validation_report.md`: 테스트 결과 기록
- `README.md`: 목적, 실행, 데이터 주의사항, 제외 기능 갱신

## 수정 후 기대 화면

- 첫 화면 상단에서 "이번 주말 바로 가능한 후보", "무료/저비용", "실내", "플랜B" 수를 확인한다.
- 추천 카드에는 프로그램명, 대상연령, 날짜, 비용, 실내/실외, 예약상태, 추정 이동시간, 추천 이유, 주의 문구가 보인다.
- `이번 주말`, `±7일`, `확장 후보`, `추후 예정` 배지로 날짜 성격을 구분한다.
- 조건 완화 퍼널에서 날짜, 자치구, 비용, 실내/실외, 관심분야 조건 완화 효과를 막대그래프로 본다.
- 추천 탐색 지도는 예약상태/추천가능성 색상으로 후보를 표시한다.
- 후보 비교에는 "가장 현실적인 선택" 자동 코멘트가 붙는다.
- 상세 + 플랜B 화면은 보호자 동반, 준비물, 취소, 노쇼, 최종 확인 필요사항을 카드형으로 보여준다.
- 정책 인사이트는 차트보다 결론 문장과 자동 요약 인사이트를 먼저 보여준다.
- 데이터 품질 패널에서 샘플 여부와 필드 품질을 확인할 수 있다.

## 완료 기준

- 기본 추천에 30일 초과 미래 후보가 포함되지 않는다.
- 예약 가능/상시 후보가 접수 예정 후보보다 우선한다.
- 대상연령, 비용, 실내 필터가 추천 결과에 반영된다.
- 추천 이유와 주의 문구가 카드/비교/상세에 표시된다.
- 이동시간은 추정 범위와 산정 방식이 명시된다.
- 조건 완화 퍼널이 앱과 `outputs/figures/06_condition_relaxation_funnel.png`에 생성된다.
- 정책 인사이트와 데이터 품질 패널이 정상 렌더링된다.
- README와 문서가 제출 설명에 맞게 갱신된다.
- `docs/validation_report.md`에 테스트 결과가 표로 기록된다.
