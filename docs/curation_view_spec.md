# 큐레이션 뷰 설계

## 목적

부모용 홈은 전체 후보 수를 보여주는 데이터 목록이 아니라, 선택 날짜에 아이와 갈 만한 곳을 빠르게 고르는 큐레이션 화면이다.

## 화면 순서

1. 오늘의 결론
2. 오늘의 추천 1곳
3. 오늘의 Top 5
4. 막혔을 때 갈 만한 곳
5. 조건을 넓히면 늘어나는 후보
6. 지금은 추천하지 않은 이유
7. 더보기

## 큰 숫자 숨김 원칙

300개 이상의 큰 숫자는 첫 화면의 핵심 메시지로 강조하지 않는다. 부모 홈의 요약 카드는 오늘의 추천, 바로 확인할 수 있는 곳, 예약 부담 낮은 대체 후보, 제외된 부적합 후보만 보여준다. 전체 데이터 수는 데이터 확인 영역에서 다룬다.

## Top 1 기준

오늘의 추천 1곳은 대상층 적합, 아이 나이 적합, 선택 날짜 또는 같은 주말, 평일 전용 아님, 마감/종료 아님, 공식 링크 존재 조건을 모두 통과해야 한다.

## Top 5 기준

Top 5는 하드필터를 통과한 후보 중 날짜 일치, 신청 상태, 예약 부담, 이동 난이도, 비용, 실내/실외 조건을 종합해 정렬한다. 나머지는 “조건에 맞는 나머지 N개 더보기” 안에 접는다.
