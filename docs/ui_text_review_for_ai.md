# 서울 데이터 기반 아이체험 큐레이터 UI 문구 검토 요청

작성일: 2026-05-13

## 1. 서비스 개요

- 이 앱은 예약 서비스가 아니라 서울 공공데이터 기반 아이체험 선택지원 시각화 도구다.
- 부모가 선택한 날짜와 조건에 맞는 아이체험 후보를 빠르게 좁히도록 돕는다.
- 실제 신청 가능 여부와 잔여석은 공식 링크에서 최종 확인해야 한다.
- 로그인, 자동예약, 결제, 알림, 사용자 DB 저장 기능은 없다.

## 2. 주요 사용자

- 주말에 아이와 갈 공공체험을 찾는 보호자
- 특히 초등 저학년 자녀를 둔 부모
- 금요일 저녁 또는 토요일 오전에 빠르게 선택하려는 사용자
- 공모전 심사 맥락에서는 데이터 출처와 한계를 확인하려는 심사자

## 3. 화면 흐름

1. 조건 입력
2. 오늘의 추천
3. 추천 후보
4. 막혔을 때 갈 만한 곳
5. 지금은 추천하지 않은 이유
6. 전체 지도
7. 이미 가본 곳 제외 설정
8. 서울시 관점

## 4. 검토 요청 기준

다음 관점으로 문구를 검토해 주세요.

- 부모가 이해하기 쉬운가?
- 너무 행정적이거나 기술적인가?
- “예약 가능”을 과도하게 확정적으로 말하지 않는가?
- 공공데이터 기반 한계를 신뢰감 있게 설명하는가?
- 버튼이 다음 행동을 명확히 안내하는가?
- 용어가 자연스러운 한국어인가?
- 시각화 공모전 제출용으로 너무 서비스 광고처럼 보이지 않는가?

## 5. 현재 문구와 대체 후보

### 메인 제목/부제

- 현재 문구: `이번 주말, 우리 아이 어디 갈까?`
- 용도: 앱의 첫 질문형 제목
- 문제점: 큰 문제는 없으나 아이체험 키워드가 제목에는 직접 드러나지 않는다.
- 대체 후보 1: `이번 주말, 아이와 어디 갈까?`
- 대체 후보 2: `이번 주말 아이체험, 어디가 좋을까?`
- 대체 후보 3: `우리 아이 주말 체험, 어디부터 볼까?`
- 검토 요청: 친근함과 명확성 중 어떤 제목이 더 나은지 검토해 주세요.

### 상단 설명

- 현재 문구: `서울 공공데이터로 날짜·나이·지역·비용 조건에 맞는 아이체험을 먼저 추려봅니다. 최종 신청 가능 여부는 공식 예약/출처 링크에서 확인하세요.`
- 용도: 서비스 목적과 데이터 한계 설명
- 문제점: “공식 예약/출처”가 다소 딱딱하다.
- 대체 후보 1: `서울 공공데이터를 바탕으로 아이 나이와 날짜, 지역에 맞는 체험 후보를 먼저 골라봅니다. 신청 가능 여부는 공식 링크에서 확인하세요.`
- 대체 후보 2: `아이 나이·날짜·지역 조건에 맞는 서울 공공체험을 먼저 추려드립니다. 최종 신청은 공식 링크에서 확인해야 합니다.`
- 대체 후보 3: `이번 주말 갈 만한 서울 공공체험을 조건에 맞춰 좁혀봅니다. 신청 가능 여부는 공식 페이지에서 확인하세요.`
- 검토 요청: 데이터 한계를 숨기지 않으면서도 첫인상이 무겁지 않게 다듬어 주세요.

### 데이터 상태 배지

- 현재 문구: `실제 API 데이터 · 업데이트 {시간}`
- 용도: 데이터 기준과 업데이트 시각 표시
- 문제점: 부모에게 `API`가 기술적으로 느껴질 수 있다.
- 대체 후보 1: `공공데이터 기준 · 업데이트 {시간}`
- 대체 후보 2: `서울시 데이터 기준 · 업데이트 {시간}`
- 대체 후보 3: `실제 공공데이터 기준 · 업데이트 {시간}`
- 검토 요청: 부모용 화면과 심사용 화면 모두에 적합한 표현을 골라 주세요.

### 필터 영역

- 현재 문구: `핵심 필터`, `아이 나이`, `학년/단계`, `출발 자치구`, `날짜`, `이동시간`, `비용`, `실내/실외`, `관심분야`
- 용도: 조건 입력
- 문제점: `출발 자치구`, `이동시간`이 다소 행정적이거나 정확한 교통시간처럼 보일 수 있다.
- 대체 후보 1: `어디서 출발하나요?`, `이동시간(추정)`
- 대체 후보 2: `출발 지역`, `갈 수 있는 이동시간`
- 대체 후보 3: `출발하는 구`, `이동 부담`
- 검토 요청: 모바일에서 짧고 이해하기 쉬운 필터 라벨을 제안해 주세요.

### 추천 결과 보기 버튼

- 현재 문구: `추천 결과 보기`
- 용도: 필터 적용 후 결과 확인
- 문제점: 큰 문제 없음
- 대체 후보 1: `추천 다시 보기`
- 대체 후보 2: `이 조건으로 보기`
- 대체 후보 3: `갈 만한 곳 보기`
- 검토 요청: 필터 적용 버튼으로 가장 자연스러운 문구를 골라 주세요.

### 오늘의 결론

- 현재 문구: `{자치구}에서 출발하는 {나이}세 아이 기준, {날짜}에 가장 추천하는 곳은 '{프로그램명}'입니다.`
- 용도: 선택 조건에 따른 대표 추천 안내
- 문제점: “가장 추천”이 다소 단정적으로 보일 수 있다.
- 대체 후보 1: `{자치구}에서 출발하는 {나이}세 아이 기준, 먼저 확인해볼 곳은 '{프로그램명}'입니다.`
- 대체 후보 2: `{날짜} 조건에서 가장 먼저 볼 만한 후보는 '{프로그램명}'입니다.`
- 대체 후보 3: `현재 조건에서는 '{프로그램명}'을 먼저 확인해보세요.`
- 검토 요청: 데이터 기반 추천이되 과도하게 확정적이지 않은 표현을 제안해 주세요.

### 오늘의 추천

- 현재 문구: `오늘의 추천 1곳`
- 용도: 대표 후보 섹션
- 문제점: “오늘”이 실제 오늘인지 선택 날짜인지 혼동 가능
- 대체 후보 1: `선택한 날짜의 추천 1곳`
- 대체 후보 2: `먼저 볼 후보 1곳`
- 대체 후보 3: `가장 먼저 확인할 곳`
- 검토 요청: 선택 날짜 기준임이 자연스럽게 드러나는 표현을 골라 주세요.

### 추천 후보 5곳

- 현재 문구: `추천 후보 5곳`
- 용도: 대표 후보 목록
- 문제점: 큰 문제 없음
- 대체 후보 1: `같이 비교할 후보 5곳`
- 대체 후보 2: `조건에 맞는 후보 5곳`
- 대체 후보 3: `추천 목록 5곳`
- 검토 요청: 부모가 “비교해볼 목록”으로 이해하기 좋은 표현을 제안해 주세요.

### 막혔을 때 갈 만한 곳

- 현재 문구: `첫 후보가 어려울 때 볼 곳`, `대체 후보`
- 용도: 첫 후보가 마감/불가일 때 볼 후보
- 문제점: “어려울 때”가 구어체라 좋지만 다소 모호할 수 있다.
- 대체 후보 1: `첫 후보가 안 될 때 볼 곳`
- 대체 후보 2: `대신 확인할 후보`
- 대체 후보 3: `막혔을 때 갈 만한 곳`
- 검토 요청: 부모에게 가장 직관적인 대체 후보 표현을 골라 주세요.

### 조건을 넓히면 늘어나는 후보

- 현재 문구: `조건을 넓히면 늘어나는 후보`
- 용도: 조건 변경 시 후보 수 변화 시각화
- 문제점: 길지만 의미는 명확함
- 대체 후보 1: `조건을 조금 풀면?`
- 대체 후보 2: `후보를 늘리는 방법`
- 대체 후보 3: `조건을 바꾸면 더 보입니다`
- 검토 요청: 차트 제목으로 가장 자연스러운 문구를 골라 주세요.

### 지금은 추천하지 않은 이유

- 현재 문구: `지금은 추천하지 않은 이유`
- 용도: 제외/마감/불일치 후보 설명
- 문제점: 큰 문제 없음
- 대체 후보 1: `조건과 맞지 않은 이유`
- 대체 후보 2: `이번 조건에서 제외한 이유`
- 대체 후보 3: `추천에서 뺀 이유`
- 검토 요청: 신뢰 장치처럼 보이되 부정적으로 들리지 않는 표현을 제안해 주세요.

### 공식 링크 안내

- 현재 문구: `공식 링크에서 확인`
- 용도: 공식 예약/출처 페이지 이동
- 문제점: 짧고 안전하지만 구체성이 부족할 수 있음
- 대체 후보 1: `공식 페이지에서 확인`
- 대체 후보 2: `공식 링크에서 신청 가능 여부 확인`
- 대체 후보 3: `출처 페이지 열기`
- 검토 요청: 예약 대행처럼 보이지 않으면서 클릭 목적이 명확한 표현을 골라 주세요.

### 예약/잔여석 주의 문구

- 현재 문구: `회차별 잔여석과 신청 가능 여부는 공식 예약/출처 링크에서 최종 확인하세요.`
- 용도: 데이터 한계 고지
- 문제점: 반복되면 방어적으로 보일 수 있음
- 대체 후보 1: `실제 신청 가능 여부는 공식 페이지에서 한 번 더 확인하세요.`
- 대체 후보 2: `회차별 잔여석은 달라질 수 있어 공식 링크 확인이 필요합니다.`
- 대체 후보 3: `이 화면은 후보 탐색용이며, 신청 가능 여부는 공식 페이지 기준입니다.`
- 검토 요청: 신뢰감을 주면서도 불안을 키우지 않는 문장을 제안해 주세요.

### 이미 가본 곳 제외 설정

- 현재 문구: `이미 가본 곳 제외 설정`
- 용도: 세션 기반 제외 조건 설정
- 문제점: “설정”이 약간 딱딱함
- 대체 후보 1: `이미 가본 곳 빼기`
- 대체 후보 2: `가본 곳은 제외하기`
- 대체 후보 3: `가본 장소/프로그램 빼기`
- 검토 요청: 저장하지 않는 임시 필터임을 자연스럽게 표현해 주세요.

### 설정 완료 후 재추천 문구

- 현재 문구: `이 설정으로 다시 추천 보기`
- 용도: 이미 가본 곳 제외 조건 적용
- 문제점: 큰 문제 없음
- 대체 후보 1: `가본 곳 빼고 다시 보기`
- 대체 후보 2: `이 조건으로 추천 다시 보기`
- 대체 후보 3: `제외하고 다시 계산하기`
- 검토 요청: 부모용으로 가장 자연스러운 버튼 문구를 골라 주세요.

### 전체 지도

- 현재 문구: `전체 지도`
- 용도: 후보 위치 확인
- 문제점: 버튼으로는 명사형
- 대체 후보 1: `전체 지도 보기`
- 대체 후보 2: `지도에서 보기`
- 대체 후보 3: `위치로 비교하기`
- 검토 요청: 버튼과 탭 제목 각각에 맞는 표현을 제안해 주세요.

### 후보 비교

- 현재 문구: `후보 비교`, `최대 3개 후보를 골라 비교하세요`, `가장 현실적인 선택`
- 용도: 후보 2~3개 비교
- 문제점: “가장 현실적인 선택”이 다소 단정적
- 대체 후보 1: `먼저 볼 만한 선택`
- 대체 후보 2: `비교상 가장 무난한 후보`
- 대체 후보 3: `현재 조건에서 눈에 띄는 후보`
- 검토 요청: 결정 보조 느낌은 살리고 확정 판단처럼 보이지 않게 다듬어 주세요.

### 상세 확인

- 현재 문구: `상세 + 대체 후보`, `이용규칙 카드`, `추천 이유와 주의할 점`, `주변/유사 대체안`
- 용도: 상세 정보와 대체 후보 확인
- 문제점: `이용규칙 카드`, `노쇼 규칙`, `주변/유사 대체안`이 다소 보고서식
- 대체 후보 1: `상세 정보와 대신 볼 곳`, `이용 전 확인할 것`, `불참/미참여 규칙`
- 대체 후보 2: `자세히 보기`, `가기 전 확인`, `대신 갈 만한 곳`
- 대체 후보 3: `후보 상세`, `예약 전 확인`, `비슷한 후보`
- 검토 요청: 상세 화면을 부모용 생활어로 다듬어 주세요.

### 서울시 관점

- 현재 문구: `서울시 관점 / 지역별 기회 보기`
- 용도: 정책/심사용 분석 탭
- 문제점: 큰 문제 없음. 분석 탭이므로 다소 정책어 허용
- 대체 후보 1: `서울시 관점`
- 대체 후보 2: `지역별 기회 보기`
- 대체 후보 3: `지역별 아이체험 격차 보기`
- 검토 요청: 공모전 심사자에게 분석 화면임이 드러나는 제목을 제안해 주세요.

### 정책 화면 분석 문구

- 현재 문구: `정책 실행 브리프`, `무엇이 후보를 줄이는가`, `우선 점검 매트릭스`, `빈 주말 캘린더`, `연령별 블라인드 스팟`, `선택가능도`, `대체 후보 충분도`, `이용장벽`
- 용도: 정책/심사용 시각화 설명
- 문제점: 부모용에는 어렵지만 정책 탭에서는 허용 가능
- 대체 후보 1: `지역별 점검 요약`, `조건별 후보 감소`, `우선 점검표`
- 대체 후보 2: `어디가 부족한가`, `어떤 조건에서 줄어드는가`, `지역 유형 보기`
- 대체 후보 3: `정책 참고 요약`, `후보 손실 분석`, `지역별 보완 우선순위`
- 검토 요청: 분석성과 이해 쉬움을 함께 만족하는 정책 탭 문구를 제안해 주세요.

### 데이터 확인

- 현재 문구: `데이터 확인`, `공공서비스예약 수집`, `문화행사 수집`, `연령 파싱 성공률`, `좌표 보유율`, `예약상태 확인율`, `공식 링크 보유율`
- 용도: 데이터 신뢰도 설명
- 문제점: 부모용에는 기술적이나 심사용으로 필요
- 대체 후보 1: `수집 기준 확인`
- 대체 후보 2: `데이터 기준 보기`
- 대체 후보 3: `데이터 수집 상태`
- 검토 요청: 심사용 신뢰도 패널로 적절한 제목과 설명을 제안해 주세요.

### 오류/빈 상태 안내

- 현재 문구: `현재 조건에서 바로 추천할 만한 후보가 없습니다.`, `추천 후보 5곳으로 보여줄 후보가 없습니다.`, `현재 조건에서는 신청 부담이 낮은 대체 후보가 없습니다.`
- 용도: 결과 없음 안내
- 문제점: 일부 문구가 딱딱함
- 대체 후보 1: `지금 조건에서는 바로 보여드릴 후보가 적습니다. 조건을 조금 넓혀볼까요?`
- 대체 후보 2: `조건을 조금 바꾸면 더 많은 후보를 볼 수 있습니다.`
- 대체 후보 3: `현재 조건에서는 추천 후보가 부족합니다. 날짜나 지역을 넓혀보세요.`
- 검토 요청: 빈 상태에서도 사용자가 다음 행동을 알 수 있게 다듬어 주세요.

### 샘플/API 모드 안내

- 현재 문구: `현재는 샘플 데이터 기반입니다. 실제 제출 전 서울시 OpenAPI 키를 연결해야 합니다.`, `샘플 데이터와 API 데이터가 섞였습니다. 캐시를 비우고 다시 수집하세요.`
- 용도: 데이터 모드 안내
- 문제점: 부모용보다는 개발/심사용에 가까움
- 대체 후보 1: `현재는 예시 데이터 기준입니다. 실제 데이터 연결 후 결과가 달라질 수 있습니다.`
- 대체 후보 2: `공공데이터 연결 상태를 확인해 주세요.`
- 대체 후보 3: `데이터 기준이 섞였습니다. 다시 수집한 뒤 확인해 주세요.`
- 검토 요청: 사용자 불안을 줄이면서 데이터 상태를 정확히 알리는 표현을 제안해 주세요.

## 6. 특히 검토가 필요한 용어

| 현재/최근 표현 | 문제 가능성 | 후보 대체어 |
|---|---|---|
| 신청 부담 낮은 대체 후보 | 의미는 맞지만 약간 딱딱함 | 신청이 쉬운 대체 후보, 확인할 일이 적은 후보 |
| 바로 확인할 수 있는 곳 | 확인의 의미가 모호할 수 있음 | 조건에 맞는 후보, 먼저 볼 후보 |
| 제외된 부적합 후보 | 부정적 어감 | 조건과 맞지 않는 후보 |
| 노쇼 규칙 | 업계 용어 | 불참/미참여 규칙 |
| 선택가능도 | 정책 지표 용어 | 갈 수 있는 후보 수 |
| 우선 점검 매트릭스 | 분석 용어 | 우선 점검표, 지역별 점검표 |
| API | 기술 용어 | 공공데이터, 서울시 데이터 |
